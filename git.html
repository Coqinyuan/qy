Git 命令

初始化本地仓库：git init

查看仓库中所有的文件状：git status

把工作目录中的文件添加到暂存区：git add 文件列表； git add *

把暂存区中的代码添加到本地仓库：git commit -m 备注信息

回滚操作：
    git reset
        --hard 回滚本地仓库的快照；使用本地仓库快照覆盖暂存区；使用本地仓库快照覆盖工作目录
        --soft 只回滚本地仓库的快照
        --mixed（默认） 回滚本地仓库的快照 并且 覆盖暂存区，但是不覆盖工作目录

删除暂存区指定的文件：git rm --cached

回滚修改操作(使用暂存区覆盖工作目录)：
    (1)git checkout -- 文件列表
    (2)git checkout . 回滚所有操作

分支：

查看分支：git branch

创建分支：git branch 分支名称

切换分支：git checkout 分支名称

创建并切换分支：git checkout -b 分支名称

合并分支：git merge 分支名称（来源分支）

删除分支：
git branch -d 分支名称（如果分支被合并过了才可以删除，否则不可以）
git branch -D 分支名称（强制删除）

远程仓库：
（1）上传代码：
  git push 远程仓库的地址 本地分支名称:远程分支名称
  git push 远程仓库的地址 本地分支名称
（2）下载代码：
  git pull 远程仓库的地址 远程分支名称:本地分支名称
  git pull 远程仓库的地址 远程分支名称
第一次下载代码一般使用如下的命令：
git clone 远程仓库的地址 [项目名称]

克隆代码：git clone 远程仓库地址 [项目名称]

更新代码：git pull 远程仓库地址 远程分支名称:本地分支名称

上传代码：git push 远程仓库地址 本地分支名称:远程分支名称

查看别名：git remote -v

添加别名：git remote add 别名名称 远程仓库地址

删除别名：git remote remove 别名名称

删除远程分支：
  git push origin --delete 分支名称
  git push origin :分支名称

.gitignore文件的作用是排除一些文件或者目录，使其不被git管理

保存现场：
  git stash
  git stash pop
